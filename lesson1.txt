أوامر أساسية لإنشاء وتكوين المستودعات
git init: يقوم بإنشاء مستودع Git جديد في الدليل الحالي.
git clone <repository_url: ينسخ مستودعًا موجودًا من عنوان URL معين إلى جهازك. 
أوامر إدارة التغييرات (العمل المحلي)
git status: يعرض حالة الملفات في دليل العمل الخاص بك، موضحًا الملفات التي تم تعديلها، والمضافة إلى منطقة الإعداد، أو غير المتتبعة.
git add <file_name أو git add .: يضيف التغييرات في ملف واحد أو كل التغييرات في المستودع إلى منطقة الإعداد (staging area) استعدادًا لعملية الالتزام (commit).
git commit -m "Commit Message": يحفظ التغييرات الموجودة في منطقة الإعداد إلى سجل الإصدارات (repository history) مع رسالة توضح محتوى الالتزام.
git diff: يعرض الاختلافات بين الإصدارات المختلفة للملفات أو بين منطقة الإعداد والمستودع. 
أوامر إدارة الفروع
git branch: يعرض قائمة بالفروع الموجودة في المستودع، ويمكن استخدامه أيضًا لإنشاء فروع جديدة.
git checkout <branch_name: ينتقل بك إلى فرع آخر محدد.
git merge <branch_name: يدمج التغييرات من الفرع المحدد إلى الفرع الحالي. 
أوامر التعامل مع المستودعات البعيدة
git remote add origin <repository_url: يضيف مستودعًا بعيدًا بعنوان URL محدد.
git push origin <branch_name: يدفع (يرفع) التغييرات المحلية إلى الفرع المحدد في المستودع البعيد.
git pull origin <branch_name: يجلب التغييرات من المستودع البعيد ويدمجها في المستودع المحلي. 
أوامر إضافية مفيدة
git log: يعرض سجل (history) الالتزامات (commits) في المستودع.
git reset: يستخدم لإلغاء التغييرات الأخيرة أو إعادة المستودع إلى حالة سابقة.
