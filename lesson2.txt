الخطوات الأساسية لرفع التغييرات إلى GitHub:
إضافة التغييرات (git add):
استخدم الأمر git add <اسم_الملف> لإضافة ملف معين إلى مساحة التجهيز (staging area)، أو git add . لإضافة جميع الملفات المتغيرة في المجلد الحالي.
تثبيت التغييرات (git commit):
بعد إضافة التغييرات، استخدم الأمر git commit -m "رسالة وصفية للتغييرات" لتثبيت هذه التغييرات في تاريخ المشروع مع رسالة تصف ما تم إنجازه.
رفع التغييرات إلى GitHub (git push):
أخيراً، استخدم الأمر git push origin <اسم_الفرع> لإرسال التغييرات التي قمت بتثبيتها إلى الفرع المحدد في المستودع البعيد (على GitHub). على سبيل المثال، git push origin main لرفع التغييرات إلى الفرع الرئيسي. 
أوامر إضافية ومفاهيم ذات صلة:
git clone:
قبل البدء، تحتاج إلى استنساخ مستودع موجود على GitHub إلى جهازك المحلي باستخدام هذا الأمر لبدء العمل على المشروع. 
git branch:
لإنشاء فروع جديدة لميزات مختلفة أو لإصلاحات معينة. 
git pull:
لجلب آخر التغييرات من المستودع البعيد إلى نسختك المحلية، وهو أمر ضروري للتعاون مع الآخرين
